Jerarquía en CSS (Restricciones)

Los navegadores aplican estilos a los documentos HTML por jerarquía basada en:

  1. Cascada
  2. Especificidad
  3. Herencia

La Cascada es el procedimiento con el que un navegador aplica los estilos. El orden de importancia o restricción es el siguiente:

  1. Origen del código
    • Reglas del User Agent (navegador)
    • Reglas del usuario (como configura el usuario el navegador)
    • Reglas del desarrollador
  2. La especificidad del selector
  3. Orden de aparición

La especificidad es el peso o fuerza restrictiva que tiene un selector.

  1. Estilo en línea 1
  2. Identificadores 0,1
  3. Clases, atributos, pseudoclases 0,0,1
  4. Etiquetas HTML 0,0,0,1
  5. !important - se salta toda la jerarquía

La herencia es la capacidad que tienen los elementos padre de transmitir a sus hijos sus mismas propiedades.

Resetear navegadores

Ante la diferente interpretación del aspecto de los elementos HTML de cada navegador, nace la necesidad de borrar o resetear todas estas propiedades nativas de los navegadores para dejar el control a los desarrolladores.
Existen en el mercado diversos archivos CSS que funcionan como reseteadores estandard, como normalize.css o reset.css.
Ver inicio del archivo indexStyle.css para más información.
